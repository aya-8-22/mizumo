# 名前：ミズモ（水＋メモ）

## 1.サービス概要
高齢者が、毎日の水分補給を習慣化するためのアプリです。
飲水記録を簡単に入力でき、カレンダーで達成状況を視覚的に確認できます。
メール通知機能により、飲水のタイミングを忘れずにリマインドします。

## 2.このアイデアはどこから生まれたか
### 2-1. きっかけとなった体験・感情
- いつ・どんな状況で：
医療機関で働いていた時、夏場になると発熱で搬送される高齢者が急増する状況を何度も目にしてきました。
特に印象的だったのは、高齢の患者様がよく「そういえば、あんまり水分とっていなかった。」と話していたことです。

- どんな気持ちになったか：
発熱があると、コロナウイルスやインフルエンザの可能性を考慮し、隔離や感染対策を行う必要があります。検査で陰性が確認されるまで、感染対策を継続しなければなりません。
その時、私は「もし水分補給を習慣化できていれば、この患者さんは熱中症にならず、医療機関の逼迫も予防できたのではないか」と強く感じました。
令和7年度の統計では、65歳以上の熱中症による救急搬送は全体の約6割（総務省 https://www.fdma.go.jp/disaster/heatstroke/items/r7/heatstroke_nenpou_r7.pdf）に及んでおり、この問題は個人の問題ではなく、医療現場全体の課題だと実感しました。

### 2-2. なぜそれが「気になった」のか
- 自分の性格、過去の経験
医療現場で働いていた経験から、「予防できる病気で医療機関が逼迫する」という状況に強い違和感を感じていました。
高齢者は体温調節機能の低下、認知機能や判断力の低下により、水分摂取を忘れやすい傾向にあります。しかし、「水分補給を忘れないようにする」という習慣化さえできれば、熱中症は予防できるはずです。

- 大切にしている価値観：
私は「予防できる病気は予防すべき」という考えを大切にしています。
特にコロナ禍以降、医療機関では感染対策の負担が増しており、「予防できる患者を減らすことで、医療現場の負担を軽減したい」という思いが強くありました。
また、医療現場で働いていた時に、「もっと早く対策できていれば」と感じる場面を何度も経験したことが、このアプリを作りたいと思った理由です。

3. 課題の整理（表面的になっていないか）
3-1. 表に見えている困りごと
どんな生活場面で困るのか：
  - 朝起きてから夕方まで、一度も水を飲まずに過ごしてしまうことがあります
  - 高齢者は「喉が渇いたら飲めばいい」と思っているが、喉の渇きを感じにくいため、気づいたときには頭痛やめまいなどの脱水症状が出ています
  - 他者が「水飲んだ？」と声掛けしても、「さっき飲んだ」と答えるが、実際には飲んでいないこともあります

何が不便なのか：
  - 水分補給を記録するアプリはあるが、記録を忘れてしまいます
  - 記録を忘れると、達成状況が正しく反映されず、アプリの価値が下がります
  - 他者が毎回声掛けするのは負担が大きいです

なぜ困るのか：
  - 水分補給を忘れると、脱水症や熱中症のリスクが高まります
  - 特に夏場は、気づかないうちに脱水状態になり、救急搬送されるケースも多いです
  - 高齢者は喉の渇きを感じにくいため、意識的に水分補給をする必要があります
  - 医療機関の逼迫にもつながります

なぜWebアプリで解決する必要があるのか：
  - メール通知により、定期的に水分補給を思い出せます
  - 記録が自動的に残り、カレンダーで達成状況を視覚的に確認できます
  - 家族が遠方にいても、達成状況を共有できます(将来的な展開)
  - スマートフォンやパソコンがあれば、どこでも使えます


3-2. 本当に解決したい課題は何か
- なぜ水分補給を忘れるのか？
  - 加齢により、喉の渇きを感じにくくなりやすいからです。

- なぜ喉の渇きを感じにくいのか？
  - 加齢により、体の水分量が減少し、喉の渇きを感じる機能が低下しているからです。

- なぜそれが問題なのか？
  - 水分不足が続くと、脱水症や熱中症などのリスクが高まり、健康を損なうからです。

- 本当に解決したい課題
  - 高齢者が、喉の渇きを感じなくても、定期的に水分補給を思い出し、習慣化できるようにすることです。

## 4. 想定ユーザーについて
### 4-1. 想定しているユーザー
年齢設定：
- 年齢：
  - 65歳〜75歳の高齢者（飲水制限ない方）。
- 理由：
  - 熱中症による救急搬送の6割が65歳以上であり、この年代にアプローチすることで、熱中症予防に大きな効果が期待できます。また、65歳〜75歳はスマートフォンやメールを日常的に使っている方が多いため、アプリの導入ハードルが低いと考えました。

生活・状況設定：
- 生活・状況：
  - 今回は飲水制限がなく、日常生活を自宅で送っている高齢者を対象とします。
- 理由：
  - 他者を介さずに、アプリの力で、定期的な飲水摂取を促すことができるためです。

ターゲットユーザーの明確化：
- 最初に登録するのは誰か：
  高齢者本人が登録します。家族がサポートすることもありますが、基本的には高齢者本人が自分で登録できることを前提とします。
- 毎日使うのは誰か：
  高齢者本人が毎日使います。メール通知を受け取り、ワンタップで記録を行います。
- 継続を支えるのは誰か：
  高齢者本人がメール通知とカレンダーの達成感によって継続します。家族は「達成状況を見守る」程度の関わり方を想定しており、毎日の声掛けやサポートは不要です。

なぜ高齢者本人が自力で継続できるのか：
  - メール通知により、定期的に水分補給を思い出せる
  - ワンタップで記録できるため、操作が簡単
  - カレンダーで達成状況を確認できるため、達成感が得られる
  - 家族の声掛けに依存せず、アプリの仕組みで習慣化できる

環境設定：
- 環境：
  - メール通知を採用します。
- 理由：
  - 高齢者の中には、LINEやスマホアプリに不慣れな方も多くいます。一方で、メールはフィーチャーフォン（ガラケー）時代から使い慣れている方が多く、スマホに機種変更した後もメールアプリは標準でインストールされているため、導入のハードルが低いと考えました。ただし、将来的にはLINE通知にも対応することで、より幅広く使ってもらえるようにしたいと考えています。

将来的な展開：
  - まずは元気な高齢者で実績を作り、その後医療機関や飲水制限のある高齢者にも展開したいと考えています。
  - 医療機関や介護施設での導入には、医師の監修や飲水制限への対応が必要になるため、まずは個人ユーザーでの実績を作ることを優先します。

### 4-2. 自分とユーザーの距離
  - 医療現場での経験から、「水分補給を忘れていた」と話す高齢者を何度も見てきました。また、自分の家族にも高齢者がいるため、「将来的に身近な人にも使ってもらいたい」という思いが強くあります。

### 4-3. 実際に使われる可能性について（需要・存在確認）
まず最初に使ってくれそうな人は誰か？
  - 自分の家族や親戚の高齢者、知人など、身近な高齢者にまず使ってもらいたいと考えています。

その人は、どんな場面でこのアプリを思い出すか？
  - 「いつ、水を飲んだかな？」と不安になったとき、またはメール通知を受け取ったときです。

今の生活の中で、代わりに使っているものは何か？
  - 家族や介護者などの他者の声掛けです。

それを置き換えてまで使う理由はあるか？
  - 家族や介護者が常に声掛けできるわけではないため、アプリで自動的にリマインドされることで、水分補給を忘れにくくなります。また、カレンダーで達成状況を確認できるため、「今日も達成できた」という小さな達成感が継続のモチベーションになります。

### 4-4. ユーザーがサービスを導入して利用するまでのイメージ（導入・継続の流れ）
どんな場面・困りごとがきっかけで、このサービスを知る・思い出すか
  - 家族や友人が「水分補給を忘れないようにするアプリがあるよ」と勧めてくれたとき、または自分自身が「最近水を飲むのを忘れがち」と感じたときです。

最初の一歩（登録・導入）で、ユーザーは何をする必要があるか
  - メールアドレスと体重を登録するだけで、複雑な設定は不要で、すぐに使い始められる。

利用を続ける理由は何か？（習慣・必要性・価値）
習慣化しやすい仕組み：
  - ールでリマインドが届くので、自然と水分補給を思い出せます。
記録が簡単：
  - メール内のリンクをタップするだけで、ワンタップで記録完了します。アプリを開く手間がないため、高齢者でも簡単に記録できます。
達成感が得られる：
  - カレンダーで達成状況を確認できるので、「今日も達成できた」という小さな達成感が継続のモチベーションになります。

離脱ポイント（どんなときに使わなくなるか）
メール通知が煩わしく感じたとき： 
  - 対策として、通知頻度をカスタマイズできる機能を追加することで、ユーザーに合わせた通知設定が可能になります。
記録を忘れてしまったとき：
  - 対策として、「記録を忘れても大丈夫」というメッセージを表示し、プレッシャーを感じさせないようにします。
達成できない日が続いたとき： 
  - 対策として、今日は達成できなくても、明日からまた頑張りましょう」といった励ましのメッセージを表示します。

### 5-2. それでも自分が作りたい理由
既存サービスがある中で、
  - なぜ満足できなかったのか
  - どこに違和感があったのか
1. 若い世代向けが中心。
  - 9歳以上の健常者を対象にしたアプリが多く、高齢者への配慮が少ない点です。
2. 操作が複雑で継続しづらい。
  - 飲料の種類が20種類以上あり、記録のたびに複数の項目を入力する必要があります。
3. 文字が小さく、視認性が低い。
  - 高齢者にとって読みにくいデザインが多い傾向にあります。
4. 達成状況が分かりにくい。
  - グラフや棒グラフで表示されるが、目標達成できたかが一目でわかりにくいです。

### 5-3. 差別化を一文で決める
私のアプリの差別化ポイント：
このアプリは、「①水分摂取をメール通知で思い出す → ②ワンタップで記録 → ③カレンダーで達成感を得る」というサイクルが、習慣化に繋がり、継続しやすくなります。

既存アプリとの違い:
| 項目 | 既存アプリ | ミズモ |
|------|-----------|--------|
| 記録方法 | アプリを開いて、飲料の種類・量を選択 | メール内のリンクをタップするだけ |
| 通知方法 | アプリ内通知(見逃しやすい) | メール通知(見逃しにくい) |
| 達成感 | グラフや数値で表示(分かりにくい) | カレンダーで⭕️❌表示(分かりやすい) |
| 継続のしやすさ | 記録を忘れると使わなくなる | メール通知で思い出せるので継続しやすい |

## 6. このサービスで提供したい価値
### 6-1. ユーザーの変化
行動や気持ちの変化：
MVP後:
1. メールの通知で水分補給を思い出せる
  - 家族の声掛けに依存せず、定期的に水分補給を思い出せる
  - 「今、水を飲もう」という行動のきっかけを提供する
2. ワンタップで記録できるため、記録が続けやすい
  - メール内のリンクをタップするだけで、記録画面に遷移できる
  - 既存アプリのように、飲料の種類や量を選ぶ必要がない
  - 記録のハードルが低いため、継続しやすい
3. カレンダーで達成状況を視覚的に確認できる
  - 「今日も達成できた」という小さな達成感が継続のモチベーションになる
  - 達成状況を一目で確認できるため、習慣化の進捗を実感できる

その先に期待できる変化:
 - 短期的変化（数週間〜数ヶ月）:
  - 水分補給が習慣化される
    - 「喉が渇いたら飲む」から「時間になったら飲む」という行動変化
    - 1日の水分補給回数が増え、目標水分量を達成できるようになる
 - 中期的変化（数ヶ月〜1年）:
  - 脱水症や熱中症のリスクが軽減される
    - 水分補給が習慣化されることで、脱水状態になりにくくなる
    - 夏場の熱中症リスクが軽減される
 - 長期的変化（1年以上）:
  - 救急搬送の軽減により、医療機関の逼迫を防ぐ
    - 予防できる病気で医療機関が逼迫するのを防ぐ
    - これはアプリ単体で実現できる価値ではなく、多くのユーザーが習慣化した結果として期待できる社会的な価値です

## 7.このアプリで実現すること
### MVPで作る機能（Must：必須）
1. ユーザー登録・ログイン機能（Devise）  
  - ユーザーごとに記録を管理
  - 登録時に「メールアドレス」「パスワード」「体重」を入力

2. 飲水記録のCRUD機能（Rails標準）
  - 水分補給を記録（日時、量）
  - ワンタップ記録（コップ1杯 = 200ml固定）

3. 体重入力と目標水分量の自動計算（Rails標準）  
  - 体重を入力すると、目標水分量が自動で計算（体重 × 30ml）

4. カレンダー表示機能（Simple Calendar）
  - 達成状況を⭕️❌で視覚的に確認

5. メール通知機能（Action Mailer + Render Cron Jobs）  
  - 1日3回（朝8時、昼12時、夕方17時）にメール通知を送信
  - メール内に記録画面へのリンクを含める
  - ユーザーがログインしていない場合は、ログイン画面に遷移する（Deviseの標準機能を使用）
  - ログイン後、記録画面に遷移する
　
  MVP版で検証したいこと:
  - メール通知が届くことで、水分補給を思い出せるか確認します。
  - メールのリンクから記録画面にスムーズに遷移できるか確認します。
  - 固定時刻の通知でも習慣化に役立つか確認します。
  - ログイン画面を経由する操作が、高齢者にとって負担になるか確認します。

---

### MVP完成後に追加する機能（Should：優先）
6. 通知時刻のカスタマイズ機能（Sidekiq + Whenever）  
  - ユーザーごとに好きな時間に飲水リマインドを設定できる
  - 通知回数の調整（1日6~10回）
  - デフォルトの通知時刻を提案（起床時、朝食時、10時頃、昼食時、15時頃、夕食時、入浴前後、就寝前）
  - ユーザーが自由に追加・削除・編集できる

7. 就寝時の達成状況通知（Action Mailer + Sidekiq + Whenever）
  - 就寝時刻に、その日の達成状況をメールで通知
  - 達成率に応じてメッセージを変える
    - 100%達成:「🎉 目標を達成しました!素晴らしいです!」
    - 80%以上:「👍 もう少しで目標達成です!明日も頑張りましょう!」
    - 80%未満:「💪 明日は目標達成を目指しましょう!」
  - ポジティブなフィードバックで達成感を得られる

8. トークン認証（自動ログイン）
  - リンクには、ユーザーを識別するトークンを含める
  - ユーザーがメールのリンクをタップすると、自動的にログイン状態になり、記録画面に遷移
  - トークンの有効期限は24時間とする
  - トークンは一度しか使用できないようにする（ワンタイムトークン）
  - トークンは推測されにくいランダムな文字列を使用する
  - ユーザーはログイン画面を経由せずに、ワンタップで記録できる

9. メール配信サービスの導入（SendGrid または Mailgun）
  - メール到達率を向上させるため、専門の配信サービスを導入
  - SPF、DKIM、DMARCの設定を行い、迷惑メールフォルダに振り分けられないようにする

## 8.このアプリの懸念点とその対策
懸念点① ：メール通知が迷惑メールフォルダに入ってしまう
何が問題になりそうか：
  - メール通知が迷惑メールフォルダに振り分けられると、ユーザーがリマインドを受け取れない可能性があります。
なぜそう思うか：
  - 定期的に送信されるメールは、スパムと判定されやすいです。
そのために考えている対策：
MVP版：
  - ユーザーがメールアドレスを登録する際に、テストメールを送信して受信確認を促します。
  - 初回登録時に「メールが届かない場合は迷惑メールフォルダを確認してください」と案内します。
Should版：
  - SendGrid や Mailgun などの信頼性の高いメール配信サービスを使用します。
  - 送信元ドメインの認証（SPF、DKIM、DMARC）を設定して信頼性を高めます。

---

懸念点② ：ユーザーが飲水記録を入力し忘れる
何が問題になりそうか：
  - 飲水記録を入力しないと、達成状況が正しく反映されず、継続のモチベーションが下がります。
なぜそう思うか：
  - 高齢者は、スマホの操作に慣れていないため、記録を忘れやすいです。
そのために考えている対策：
MVP版:
  - 飲水量：コップ1杯200mlとします。
  - 記録入力を簡単にします（ワンタップで記録完了）。
  - メール通知に「記録はこちらから」というリンクを含めます。
Should版：
  - 就寝時間前に「今日の記録を確認しましょう」というリマインドメールを送ります。
Could版（将来的に追加）：
  - 設定画面で「コップ1杯の量」を選択できるようにします。
  - 選択肢：150ml / 200ml / 250ml など。
  - 記録時は、設定した量が自動的に記録されます。

---

懸念点③ ：目標水分量の計算が実態と合わない可能性
何が問題になりそうか：
  - 体重だけで目標水分量を計算すると、個人の生活習慣や健康状態に合わない可能性があります。
なぜそう思うか：
  - 運動量、気温、持病などによって必要な水分量は変わるためです。
そのために考えている対策：
MVP版：
  - 「体重 × 30ml」というシンプルな計算式を採用します。
  - 「医師に相談してください」という注意書きを表示します。
将来的な展開：
  - 運動量や気温を考慮したカスタマイズ機能を追加します。
