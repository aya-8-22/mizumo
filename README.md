## サービス概要
水分摂取を管理するタイマーアプリ。
日常生活から医療・介護現場まで幅広く利用可能。

## このサービスへの思い・作りたい理由
水分摂取は人間にとって、欠かせない要素の一つです。
しかし、意識しないと、忘れていることってありませんか。
それを解決するのがこのアプリです。
このアプリでは、水分摂取を習慣化させるために、音やバイブを使い、強制的に水分摂取を促していきます。

## ユーザー層について
- 水分補給を習慣化したい一般ユーザー
- 熱中症予防などの予防医療を行いたい一般ユーザー
- 高齢者の水分管理をサポートする介護施設
- 患者の水分管理や下剤などの大量内服が必要な医療現場
- 疾患により飲水制限があるユーザー

## サービスの利用イメージ

### 朝の準備(基本的には一度入力したら保存される)
1. アプリを起動し、体重を入力(例:60kg)
2. 自動的に1日の必要水分量が計算される(例:1800~2400ml)
3. 起床時間や就寝時間、飲みたい回数を設定(例:6~8回)

### 日中の利用
- 設定した時間になると、スマホから音やバイブで通知
- 水を飲んだら、通知を消せば、飲水をしたことになる(時間的負担を軽減するため)
- 視覚的に進捗が確認できる(例:コップのアイコンが埋まっていく)

### 夜の振り返り
- 1日の摂取量を確認
- 達成度に応じてフィードバック
- 継続日数が記録される

### 医療・介護現場での利用(将来的に)
- 患者や利用者ごとに個別の水分量を設定
- 複数人の管理が可能
- 記録を印刷・共有できる

## サービスの差別化ポイント・推しポイント
健常者の予防医療だけでなく、疾患により水分管理が必要な方にも使用できる仕様にしました。
幅広いユーザーが使用できるように、見やすく操作しやすいのがポイントです。

## MVP機能(最初に実装する機能)
- ユーザー登録・ログイン機能
- 体重入力による必要水分量の自動計算(体重(kg) × 30~40 ml = ml)
- 疾患によって体重からではなく飲水量を直接設定できる機能
- **手動で飲水記録をつける機能** (ボタンを押したら記録)
- 1日の摂取記録の表示(今日飲んだ量、目標達成率)

## 使用技術
- **バックエンド**: Ruby on Rails 7.x
- **フロントエンド**: Hotwire(Turbo + Stimulus)
- **データベース**: PostgreSQL
- **認証**: Sorcery
- **インフラ**: Render
- **通知** (将来的に): Web Push Notification API

## 画面構成(予定)

### 一般ユーザー向け
- **ホーム画面**: 今日の進捗を表示(飲んだ量、目標達成率、次の飲水時間)
- **設定画面**: 体重・飲水回数・時間などを入力
- **記録画面**: 過去の記録を表示(カレンダー形式)
- **通知設定画面**: 音やバイブの設定(将来的に)

### 疾患を抱えたユーザー向け
- **設定画面**: 医師が指定した水分量を直接入力
- **記録画面**: 医療従事者と共有できる形式(将来的に)
